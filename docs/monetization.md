## 0. 현재 구현 — 쿠팡 파트너스

### 어떻게 구현되어 있나
- **`AffiliateProperties`** (record-based) + **`AffiliateLinkBuilder`** — 쿠팡 검색 URL에 tracking-id 자동 wrap
- **`application.yaml`**: `affiliate.coupang.tracking-id` (env `COUPANG_TRACKING_ID`)
- **`ingredients/list.html`** 의 "쿠팡 검색" 컬럼 — tracking-id 비어 있으면 일반 검색 URL로 fallback
- 사용자 클릭 시 쿠팡으로 이동 → 24시간 내 구매 시 수수료 발생 (쿠팡 파트너스 표준)

### 왜 쿠팡 파트너스를 먼저?
- **이 프로젝트의 자연 fit**: 사용자가 재료를 보고 → "어 사야지" → 클릭 → 구매. 추천이 강요가 아니라 정보의 연장.
- **트래픽 작아도 즉시 시작 가능**. AdSense는 승인까지 시간 걸리지만 쿠팡은 가입 즉시 활성.
- **기존 Naver 연동과 호환**: 가격 비교 컨텍스트와 충돌 X.

### 한계
- 수수료율이 낮음 (식자재 카테고리 평균 1-3%)
- 사용자가 "쿠팡에서 구매 의도가 명확한 순간"에만 효과 — 우리 사이트의 1차 가치(원가 계산)와는 거리 있는 행동

## 1. 단기 (3-6개월) — Google AdSense

### 도입 계획
- **승인 조건**: 자체 콘텐츠(레시피) 일정 수 + 트래픽 누적 + 정책 준수(개인정보처리방침, 광고 표시 명시)
- **광고 영역 (의도적 제한)**: 사용자 경험 보호 차원에서 광고 위치는 강하게 제약
  - ✅ 허용: 홈 페이지 카드 사이, 검색 결과 페이지 하단
  - ❌ 금지: **레시피 상세(detail.html) 본문 안**, 재료 테이블/원가 계산 섹션 근처, 댓글 영역
  - 사용자가 "정보를 얻으러 온 순간"에 광고는 노이즈. 신뢰 손상 비용 > 수익.
- **반응형**: 모바일 인터스티셜은 안 쓰기. 사용자 이탈 직결.

### 측정 지표
- 페이지뷰 대비 광고 노출률, CTR, RPM
- **광고 도입 전/후 평균 세션 시간 비교** — 떨어지면 영역 축소 신호

## 2. 단기 (3-6개월) — 제휴 채널 확장

### 후보 채널 우선순위
1. **네이버 쇼핑 페이 (네이버 어필리에이트)** — 이미 Naver API로 ingredient 받아오니 자연 연결. 단가 비교의 정확도 = 우리의 신뢰 지표.
2. **11번가 어필리에이트** — 식자재 카테고리 강함, 쿠팡과 가격 차이 자주 발생 → "최저가 비교" 컨셉 강화 가능
3. **마켓컬리** (정규 어필리에이트 프로그램 있을 시) — 프리미엄 식자재 사용자 대상

### 구조 변경
- `AffiliateLinkBuilder` 인터페이스 추출 → `CoupangAffiliateLinkBuilder`, `NaverAffiliateLinkBuilder`, ... 다중 채널
- ingredient 목록 + 레시피 상세에 "어디서 살까" 다중 채널 비교 표 표시 가능

## ∞. 장기 비전 후보 — B2B 외식업 SaaS

### 왜 이게 후보인가
CLAUDE.md의 **핵심 인사이트** 인용:
> "원가 계산은 보통 업소용 대용량(예: 밀가루 20kg) 기준이므로, 제품 가격이 아니라 1g(또는 1ml)당 가격으로 정규화해서 저장 → 어떤 단위 제품이든 공정 비교 가능."

이 인사이트는 사실 **B2B 외식업 원가 계산기**의 PMF 신호다. 개인 사용자(B2C)는 "고기 200g"으로 사니까 1g당 가격이 큰 의미 없는데, **카페/베이커리/식당 사장은 매월 밀가루 20kg를 정기 매입하니 단가 비교가 매출에 직결**된다.

### 가설적 제품 형태
- 매장 단위 계정 (다인 협업: 사장 + 직원이 메뉴/원가 공동 관리)
- 매입처 견적 비교 (쿠팡/B2B 마트/도매업체 등 가격 입력 → 자동 비교)
- 메뉴별 마진 계산 + 가격 변동 알림 (밀가루가 10% 오르면 영향받는 메뉴 자동 식별)
- 단가: 월 2-5만원/매장 (개인 구독의 10배 — 매출에 직결되니 정당화)
- 매출 규모: 1,000 매장 × 월 3만원 = 월 3,000만원 (개인 구독보다 ARPU 10배+)

### 왜 지금은 안 하나
- **고객 발굴 채널이 B2C와 완전 다름** (인스타/유튜브가 아니라 영업/소상공인 커뮤니티/세무사 추천 등)
- **제품 요구사항도 다름** — 견적서 PDF, 매입처 협상 데이터, 회계 연동
- **현재 코드를 B2C 허브 컨셉으로 짜둔 상태라 점진적 추가가 아닌 분리/Pivot이 적절**

### 재검토 트리거 (6-12개월 뒤)
- B2C 트래픽이 정체되면 → B2B Pivot 진지하게 검토
- B2C 사용자 중 "사장님" 비율이 우연히 늘면 → B2B 기능 일부 통합
- 그 사이 B2B 영역에 강한 경쟁자가 자리잡으면 → 진입 포기

## 의도적으로 안 하는 것

| 모델 | 안 하는 이유 |
|---|---|
| **인터스티셜 광고 (전면 광고)** | 모바일 이탈률 직격. 신뢰 손상 비용 > 수익 |
| **레시피 잠금 → 결제 강요** | 콘텐츠 허브 컨셉 자체를 깨뜨림. 검색 노출 + SEO 손상 |
| **이메일 광고 / 푸시 광고** | 옵트인 안 받은 채널 광고는 스팸. 브랜드 신뢰 자산 손상 |
| **개인정보 판매** | 절대 X. 법적 리스크 + 윤리 위반 |
| **암호화폐/NFT 토큰화** | 현재 트렌드 사이클 외, 본질 가치와 무관 |

## 핵심 철학

1. **사용자 신뢰 자산 > 단기 수익**. 광고/제휴는 정보의 연장이어야지 강요가 되면 안 됨.
2. **차별화 = 데이터 품질**. 광고 매출보다도 "이 사이트가 가장 정확하다"가 중장기 PMF.
3. **모델은 항상 옵트아웃 가능해야**. 프리미엄 도입 시에도 무료 사용자가 "광고만 빼고 무료 사용 가능"한 옵션을 제공할지 결정 필요 (Open Q).
